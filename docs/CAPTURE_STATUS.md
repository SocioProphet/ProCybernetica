# ProCybernetica Capture Status

The immediate objective is to make the public GitHub repository self-contained enough to build from the existing Drive corpus. Implementation should follow capture, not precede it.

## Capture mode rule

Drive is the source archive. GitHub is the codification target.

Do not modify Drive source documents during capture unless there is a deliberate editorial, archival, or program-management reason.

Do not treat early schemas or runtime scaffolding as canonical until the source corpus has been captured and reconciled.
